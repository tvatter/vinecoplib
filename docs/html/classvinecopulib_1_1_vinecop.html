vinecopulib::Vinecop class

A class for vine copula models.

A vine copula model is characterized by the structure matrix (see TriangularArray) and the pair-copulas.

boost::property_tree::ptree vinecopulib::Vinecop::to_ptree() const

converts the copula into a boost::property_tree::ptree object.

Returns the boost::property_tree::ptree object containing the copula.

The ptree object contains two nodes : "matrix" and "pair copulas". The former is encodes the R-Vine structure and the latter is a list of child nodes for the trees ("tree1", "tree2", etc), each containing a list of child nodes for the edges ("pc1", "pc2", etc). See Bicop::to_ptree() for the encoding of pair-copulas.

void vinecopulib::Vinecop::select_all(const Eigen::MatrixXd& data, const FitControlsVinecop& controls = FitControlsVinecop())

automatically fits and selects a vine copula model.

Parameters
data nxd matrix of copula data.
controls the controls to the algorithm (see FitControlsVinecop).

Selection of the structure is performed using the algorithm of Dissmann, J. F., E. C. Brechmann, C. Czado, and D. Kurowicka (2013). Selecting and estimating regular vine copulae and application to financial returns. Computational Statistics & Data Analysis, 59 (1), 52-69.

Eigen::MatrixXd vinecopulib::Vinecop::inverse_rosenblatt(const Eigen::MatrixXd& u, const size_t num_threads = 1) const

calculates the inverse Rosenblatt transform for a vine copula model.

Parameters
u matrix of evaluation points.
num_threads the number of threads to use for computations; if greater than 1, the function will be applied concurrently to num_threads batches of u.

The inverse Rosenblatt transform can be used for simulation: the function applied to independent uniform variates resembles simulated data from the vine copula model.

If the problem is too large, it is split recursively into halves (w.r.t. n, the number of observations). "Too large" means that the required memory will exceed 1 GB. An examplary configuration requiring less than 1 GB is , .
